# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Is

This is an **Obsidian vault** ("My Second Brain") — a personal knowledge management system, not a software project. It contains markdown notes organized using the PARA method (Projects, Areas, Resources, Archives).

## Vault Structure (PARA Method)

- `00 - Inbox/` — Quick capture, uncategorized. Default destination when unsure.
- `01 - Projects/` — Active projects with deadlines and goals.
- `02 - Areas/` — Ongoing responsibilities (Dev, Finanças, Saúde, etc.).
- `03 - Resources/` — Reference knowledge organized by topic (Dev, Ciência, Games, etc.).
- `04 - Daily Notes/` — One note per day (`YYYY-MM-DD.md`), operational log.
- `05 - Archives/` — Completed or inactive content.

## Writing Conventions

- **Language:** Notes are written in Brazilian Portuguese (pt-BR).
- **Atomic notes:** One idea per note, clear and descriptive titles.
- **Internal links:** Use Obsidian `[[wikilinks]]` to connect notes. Links matter more than folder hierarchy.
- **Tags:** Use hashtags for categorization (e.g., `#ai #llm #local`).
- **Metadata:** Notes may include a header block with Date and Tags.
- **Grow organically:** Don't create subfolders before they're needed.

## Regras de Escrita

- Toda documentação, comentário e nota deve ser em **português brasileiro**.
- Use linguagem simples e direta, fácil de entender.
- Quando fizer sentido no contexto, inclua uma **analogia ou metáfora do mundo real** para facilitar o entendimento.

## Working With This Vault

- When creating new notes, place them in `00 - Inbox/` unless a clear category exists.
- Preserve existing `[[wikilinks]]` when editing notes.
- The `.obsidian/` directory contains Obsidian app configuration — avoid modifying it.
- Backup is handled via **Obsidian Git** plugin syncing to a private GitHub repo.
